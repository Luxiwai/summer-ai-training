# Claude Code 常用命令速查表

第一周（五子棋项目）实战积累。

---

## 一、基础命令（在 Claude Code 中输入）

| 命令 | 作用 |
|------|------|
| `/init` | 分析项目并生成 CLAUDE.md，后续对话自动加载为上下文 |
| `/clear` | 清空当前对话，释放上下文窗口 |
| `/compact` | 压缩对话历史，保留关键摘要，避免 token 溢出 |
| `/help` | 查看所有可用命令 |
| `/config` | 修改 Claude Code 配置（主题、模型等） |
| `/cost` | 查看当前会话 token 用量和费用 |

**在对话中直接让 Claude 做的事情：**

| 说法 | 效果 |
|------|------|
| "阅读每个 py 文件，告诉我每个文件的功能" | 触发全部文件阅读 + 分析总结 |
| "列出冗余代码、重复逻辑、未使用的变量/函数/导入" | 代码审查（死代码检测） |
| "指出代码中的不规范处、潜在隐患、隐形 Bug" | 代码审查（质量/风险） |
| "找出项目的框架问题，给出重构方向和意见" | 架构审查 |
| "根据我的要求对项目进行修改：1.xxx 2.xxx" | 批量重构（带编号的需求列表） |
| "将代码中的注释改为中文" | 批量文本替换（指定源语言→目标语言） |
| "生成单元测试" | 自动生成 test_game.py，基于 unittest |
| "运行测试" | 执行 `python -m unittest` |
| "据此修改 CLAUDE.md 中的测试方式模块" | 根据新测试文件同步更新文档 |

## 二、最有效的提示词模板

### 模板 1：代码审查（一次性全面诊断）

```
阅读每个[python]文件 告诉我每个文件的功能是什么。
列出所有文件的功能、文件之间的依赖关系、函数或跨文件调用关系。
标记冗余代码、重复逻辑、没有使用的变量或函数或导入。
指出代码中的不规范处、潜在隐患、隐形Bug。
找出项目的框架问题，给出重构方向和意见。
```

### 模板 2：批量重构（带优先级编号）

```
根据我的要求对项目进行修改
1.消除未使用的变量、函数、废弃代码路径
2.优化整合重复部分
3.修复Bug并添加平局检测
4.优化性能以及硬编码耦合问题
5.解决字符串魔法值问题
6.解决框架问题并进行重构
```

### 模板 3：功能添加（明确技术选型）

```
添加[pygame]库，运行[run.py]时弹出窗口，
窗口为[10*10]的棋盘，用户可交替下[黑白棋]，
先下[黑棋]，输赢评判规则不变。
```

### 模板 4：生成 CLAUDE.md（项目知识沉淀）

```
/init
```

或手动指定内容：

```
在CLAUDE.md文件中加入：
1.技术栈
2.代码规范
3.常用命令
4.测试方式
5.项目结构
```

### 模板 5：生成单元测试

```
对此项目生成单元测试
```

Claude 会自动：识别可测试的纯逻辑层 → 选择 unittest 框架 → 按功能分组 → 覆盖边界 case。

### 模板 6：注释语言切换

```
将代码中的注释改为[中文]
```

## 三、我的小窍门

### 窍门 1：用需求编号驱动多步骤任务

给 Claude 的需求加上数字编号（`1.` `2.` `3.`），它会自动创建 TodoWrite 任务列表逐项推进，不会遗漏。例如：
```
1.消除未使用的变量  2.优化重复部分  3.修复Bug  4.优化性能  5.解决魔法值  6.框架重构
```

### 窍门 2：先审查，后重构

顺序不要乱：**先让 Claude 全面审查代码**（找出所有问题），再一次性给出重构指令。如果跳过审查直接重构，Claude 可能只改表面问题，深层 Bug（如双向判胜漏判）会被忽略。

### 窍门 3：用 `/init` 沉淀项目知识

每完成一轮重构，重新跑一次 `/init` 或手动更新 CLAUDE.md。后续对话中 Claude 会自动加载这些上下文，避免重复解释项目结构。

### 窍门 4：测试让 Claude 自己跑

生成单元测试后，直接说"运行测试"让 Claude 执行并修复失败项。不要自己复制粘贴到终端——Claude 会读输出、定位失败原因、自动修改代码，迭代直到通过。

### 窍门 5：改动后立即验证

每次 Claude 改完文件，追加一句"验证一下"或"测试是否通过"。Claude 会跑语法检查 + 单元测试，第一时间发现引入的问题。本项目在重构过程中发现了测试脚本自身的 bug（切换玩家后检查了错误的字符串）。

### 窍门 6：Claude.md 和 cheatsheet 分开写

- **CLAUDE.md** → 给 Claude 看的技术文档（架构、命令、规范）
- **claude-cheatsheet.md** → 给自己看的速查表（模板、踩坑、窍门）

两个文件定位不同，一个面向 AI，一个面向人类。

### 窍门 7：重构后检查旧文件是否可删除

重构完新旧架构并存是常见的混乱源。重构完成后对 Claude 说"检查还有哪些文件可以删除"，或者直接让它清理。本项目从 5 个文件精简到 4 个（删了 `variables.py` `board.py` `win.py` `play.py`，新建 `constants.py` `game.py` `renderer.py`）。
